Getting Started with NullHandshake

About NullHandshake

NullHandshake is a powerful wireless network post-exploitation framework designed for security researchers and penetration testers. It provides a unified interface for various WiFi-based attack techniques through its plugin architecture.

The framework is built around a modular design, allowing you to leverage different attack vectors through specialized modules:

WiFi Recon Credential Harvester Evil Twin WPA Handshake
Disclaimer

This tool is intended for use in controlled environments with explicit authorization. Unauthorized access to computer networks is illegal and punishable under law.

Quick Start Guide
1. Choose a Module

Start by selecting a module from the dashboard based on your objective:

  • WiFi Recon - For scanning and analyzing wireless networks
  • Credential Harvester - For extracting stored WiFi credentials
  • Evil Twin - For creating rogue access points
  • WPA Handshake - For capturing and cracking WPA handshakes
2. Configure Your Interface

Most modules require selecting a wireless interface. Connect a compatible wireless adapter and select it in the module interface.

list_interfaces
set_interface wlan0
3. Run Commands

Each module has specific commands. Use the console interface or quick command buttons to execute operations.

4. Review Results

Results will be displayed in the console output. You can save or export data for further analysis.

Tips
  • Use the WiFi Scan page for a quick overview of nearby networks
  • Each module has detailed documentation on its page
  • For better results, use wireless adapters that support monitor mode
  • Save scan results regularly when conducting field assessments

Example Scenarios

Network Reconnaissance
Scenario: Identifying All Networks in Range

This example demonstrates how to scan for and catalog all wireless networks in your vicinity.

  1. Launch the WiFi Recon module
  2. Check available interfaces and set your wireless adapter:
    3. list_interfaces
    set_interface wlan0
  3. Start scanning and enable channel hopping:
    4. start_channel_hopping
    scan
  4. After a few minutes, view the discovered networks:
    5. show_networks
  5. Save the results to a file:
    6. save_results scan_results.txt
Outcome:

A comprehensive list of networks with details about their security settings, signal strength, channels, and BSSIDs that can be used for further analysis.

Password Recovery
Scenario: Recovering Forgotten WiFi Passwords

This example shows how to recover WiFi passwords that are stored on your system.

  1. Launch the Credential Harvester module
  2. Extract stored credentials:
    3. extract_credentials
  3. View the extracted passwords:
    4. show_credentials
  4. Save the credentials to a file:
    5. save_credentials wifi_passwords.txt
Outcome:

A list of all WiFi networks and their corresponding passwords that are stored on your system, allowing you to recover forgotten credentials.

This functionality works across Windows, macOS, and Linux platforms, retrieving stored credentials from the system's password storage.
WPA Handshake Capture
Scenario: Capturing WPA Handshakes for Password Recovery

This example demonstrates how to capture WPA handshakes that can be used for offline password recovery.

  1. Launch the WPA Handshake module
  2. Set up your wireless interface:
    3. list_interfaces
    set_interface wlan0
  3. Scan for target networks:
    4. scan_for_targets
  4. Set a target network:
    5. set_target 00:11:22:33:44:55
  5. Start capturing handshakes:
    6. capture_handshake
  6. After capturing, export the handshake:
    7. export_handshake wpa_capture.cap
Outcome:

A captured WPA handshake that can be used with offline password cracking tools to recover the network password.

Evil Twin Attack
Scenario: Setting Up a Rogue Access Point

This example shows how to create a rogue access point that mimics a legitimate network for security testing.

  1. Launch the Evil Twin module
  2. Configure your wireless interface:
    3. list_interfaces
    set_interface wlan0
  3. Scan for potential target networks:
    4. scan_for_targets
  4. Select a target network to clone:
    5. set_target 00:11:22:33:44:55
  5. Set up the rogue access point:
    6. setup_ap
  6. Start the access point:
    7. start_ap
  7. Monitor for captured credentials:
    8. show_captured_credentials
Outcome:

A functioning rogue access point that appears identical to the target network, which can be used to test security awareness and the effectiveness of user training.

This functionality should only be used in controlled environments with proper authorization for security testing purposes.

  • aircrack-ng
  • hostapd
  • dnsmasq
  • iptables
Hardware Requirements
Wireless Adapters

For the best experience, use a wireless adapter that supports monitor mode and packet injection. Recommended adapters include:

  • Alfa AWUS036ACH (802.11ac)
  • Alfa AWUS036NHA (802.11n)
  • TP-Link TL-WN722N (V1 only)
  • Panda PAU09
Minimum System Specifications
  • CPU: Dual-core processor
  • RAM: 4GB minimum, 8GB recommended
  • Storage: 1GB free space
  • Network: At least one compatible wireless adapter
For more intensive operations like WPA handshake cracking, a multi-core processor and additional RAM will significantly improve performance.

Frequently Asked Questions

NullHandshake is a modular wireless network post-exploitation framework designed for security researchers and penetration testers. It provides tools for:

  • Scanning and analyzing wireless networks
  • Extracting stored WiFi credentials from various operating systems
  • Creating rogue access points (Evil Twin attacks)
  • Capturing WPA handshakes for offline password recovery

The framework is designed to be modular, extensible, and easy to use through both a command-line and graphical interface.

For basic functionality like credential harvesting, any built-in wireless adapter will work. However, for advanced features like WiFi scanning, Evil Twin attacks, and WPA handshake capturing, you'll need a wireless adapter that supports monitor mode and packet injection.

Recommended adapters include:

  • Alfa AWUS036ACH
  • Alfa AWUS036NHA
  • TP-Link TL-WN722N (V1 only)
  • Panda PAU09

These adapters have good driver support and are compatible with the Atheros, Realtek, and Ralink chipsets that support the required features.

NullHandshake, like any security tool, should only be used for legitimate purposes such as:

  • Security assessments with explicit authorization
  • Testing your own networks
  • Educational purposes in controlled environments

Unauthorized access to computer networks, including wireless networks, is illegal in most jurisdictions and can result in severe penalties. Always ensure you have proper authorization before testing any network that you don't own.

NullHandshake offers two main methods for WiFi password recovery:

1. Extracting stored passwords (Credential Harvester)

If the password is already stored on your system:

  1. Launch the Credential Harvester module
  2. Run the extract_credentials command
  3. View the results with show_credentials
2. Capturing and cracking WPA handshakes

For networks you don't have access to:

  1. Launch the WPA Handshake module
  2. Select your interface and target network
  3. Capture a handshake with capture_handshake
  4. Export the capture file with export_handshake
  5. Use external tools like Hashcat to perform offline cracking of the handshake
The success of password recovery depends on various factors including encryption strength and password complexity. Always ensure you have proper authorization before attempting to recover passwords for networks.
